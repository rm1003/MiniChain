1. Introdução

    Este trabalho apresenta uma implementação de um sistema de moeda virtual
com tentativa de aproximar com a ideia de BitCoins, chamado  de MiniCoins,
utilizando o sistema cliente-servidor com uso de Blockchain para garantir a
integridade e segurança das transações financeiras dos clientes. O sistema é
composto por um servidor central que gerencia a Blockchain personalizada e
múltiplos clientes que interagem com o sistema por meio de uma interface de
linha de comando no terminal.

2.2 Componentes

    -> Servidor: Genrenciamento da Blockchain, autenticação dos usuários e
processa transações.

    -> Cliente: Interface para o usuário realizarem operações financeiras, como
depositar, retirar, ver saldo, transferir para outro usuário e finalizar o client.

    -> Blockchain: Estrutura de dados imutáveis para registro de transações.

    -> Sockets: Comunicação em rede usando TCP/IP.


2.3 Algumas Características do Projeto

Blockchain:
    - Implementação de uma lista ligada de blocos (registros).
    - Cada bloco contém informações:
        -> Hash atual
        -> Hash anterior
        -> Id do cliente
        -> Id do destino
        -> Valor
        -> Tempo
        -> Tipo da transação
        -> Ponteiro para próximo bloco
    - Uso do algoritmo SHA-256 para geração de hashes (criptografia).

Funcionalidade do Sistema Implementado:
    - Autenticação de usuário com nome e senha.
    - Operações de depósito, retirada e transferência entre clientes (usuários).
    - Consulta de saldo atual (calculado percorrendo desde o primeiro bloco da
Blockchain)
    - Registro do tempo de cada transação.
    - Verificação de integridade da blockchain quando insere um novo registro.
    - Validação de saldo antes de transações.

4. Resultados Obtidos

    Obtivemos uma aplicação Cliente Servidor que possui uma implementação 
simplificada de uma Blockchain, verificando a integridade da Blockchain e de suas
transações para cada nova operação e novo bloco a ser inserido.
    O servidor suporta múltiplos clientes "ao mesmo tempo" e possui uma lógica
interna que atribui um ID para cada cliente e seu username, dessa forma, enquanto
o servidor se mantiver em execução, um cliente pode encerrar sua conexão e se conectar
novamente em outro momento e ainda acessar seu saldo de MiniCoins(MC).
